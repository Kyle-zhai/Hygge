# Platform V2: Sidebar + General Discussion Mode

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Transform Hygge from a single-purpose product evaluation tool into a multi-mode discussion platform. Add a Gemini-style sidebar for navigation, keep product evaluation as the flagship mode, and add a general topic discussion mode with dynamic dimensions and AI-generated personas.

**Architecture:** The sidebar replaces the current header-only navigation, housing discussion modes, history, and settings. Both modes share the same Worker pipeline but diverge at prompt/dimension level. The `evaluations` table gains a `mode` field. The front-end report views become data-driven instead of hardcoded.

**Tech Stack:** Next.js 16, React 19, Shadcn/UI, Tailwind CSS v4, Framer Motion, Supabase, BullMQ, Claude SDK, next-intl

### Risk Mitigation

- **Product mode is never broken.** All changes are additive — product mode always falls back to existing hardcoded behavior when `topic_classification` is null.
- **Dynamic dimensions quality.** The classify-topic prompt needs careful tuning. Start with a constrained set of topic types and iterate.
- **Report rendering.** The 1550-line report-text-view.tsx is the riskiest change. Approach: create a dynamic dimension renderer alongside the existing hardcoded one, switch based on data presence.
